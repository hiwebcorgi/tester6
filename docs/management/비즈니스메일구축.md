# [SETUP] corg.app 비즈니스 이메일 무상 구축 가이드 (Cloudflare + Gmail)

## 1단계: 도메인 네임서버 연결 (Cloudflare)
- corg.app의 네임서버를 Cloudflare로 이전합니다. (도메인 관리의 표준이자 보안의 핵심)
- Cloudflare 관리 화면에서 [Email] -> [Email Routing] 메뉴로 진입합니다.

## 2단계: 수신 설정 (Catch-all / Routing)
- [Destination addresses]: 실제로 메일을 받아볼 사장님의 개인 지메일(ex: webcorgi@gmail.com)을 등록합니다.
- [Custom addresses]: `master@corg.app`을 생성하고 대상 주소를 위 지메일로 연결합니다.
- 결과: 이제 누군가 `master@corg.app`으로 메일을 보내면 사장님의 지메일로 즉시 전달됩니다. (수신 비용 $0)

## 3단계: 발신 설정 (SMTP Alias) - 가장 중요
사장님이 답장을 보낼 때도 `master@corg.app`으로 보이게 하는 마법입니다.
- 지메일 [설정] -> [계정 및 가져오기] -> [다른 주소에서 메일 보내기] -> [다른 이메일 주소 추가] 클릭.
- 이름: 웹코기 원팩 (또는 사장님 성함)
- 이메일 주소: master@corg.app
- SMTP 서버: smtp.gmail.com / 포트: 587
- 사용자 이름: 사장님의 지메일 주소 (ex: webcorgi@gmail.com)
- 비밀번호: 지메일 설정에서 생성한 [앱 비밀번호 (App Password)] 입력.
- 결과: 지메일에서 메일을 쓸 때 보낸 사람을 `master@corg.app`으로 선택할 수 있습니다. (발신 비용 $0)

## 4단계: 신뢰도 최적화 (SPF/DKIM/DMARC)
- Cloudflare가 자동으로 잡아주는 DNS 레코드를 승인합니다.
- 이 설정을 통해 사장님의 메일이 스팸함으로 빠지지 않고 고객의 받은편지함에 안전하게 안착합니다.
